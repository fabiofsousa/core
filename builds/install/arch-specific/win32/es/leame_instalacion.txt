Firebird Database Server $MAJOR.$MINOR.$RELEASE
==================================================

  ** IMPORTANTE **

  La estructura en disco de los archivos Firebird 
  (ODS) ha cambiado respecto a la versión 1.5. Lea
  las siguientes notas relacionadas con la nueva 
  estructura y su instalación sobre versiones 
  anteriores de Firebird.

  ** ********** **
  
Este documento es una guía para instalar este paquete
de Firebird $MAJOR.$MINOR.$RELEASE en la plataforma Win32. Estas notas
hacen referencia al paquete de instalación en sí, más
que a Firebird $MAJOR.$MINOR.$RELEASE en general.

Antes de instalar
-----------------

Se recomienda que DESINSTALE cualquier versión
anterior de Firebird o Interbase antes de 
instalar este paquete. Es particularmente
importante que verifique que fbclient.dll y gds32.dll
hayan sido eliminados de <system32>. 


Nueva ODS 11.0
--------------

Firebird $MAJOR.$MINOR introduce una nueva versión (11.0) 
de la estructura en disco de archivos Firebird (ODS),
que es completamente incompatible con la versión 11.0
utilizada en las versiones alpha.

Si está instalando esta versión de Firebird $MAJOR.$MINOR por
primera vez sobre una instalación previa de Firebird 2 
alpha se recomienda que lo haga en una carpeta 
diferente. Aunque la desinstalación no elimina
la base de datos de seguridad existente, ésta 
utiliza una estructura (ODS) incompatible, y no
podrá conectarse al servidor.


Instalación en sistemas con Firebird 1.5
----------------------------------------
La base de datos de seguridad ha cambiado. La nueva
base de datos se llama security2.fdb y usa un 
algoritmo de encriptación de contraseñas mejorado. 
Puede obtener más información sobre las nuevas
características de seguridad en las notas de la 
versión.

La nueva estructura de bases de datos y los cambios
en la encriptación generan varios problemas 
relacionados con la migración de sitios que tengan
muchos identificadores de usuario (user ids).
El instalador no intentará realizar esta migración, 
la cual debe ser realizada manualmente. Una posible
técnica se describe en misc/upgrade/security/. 


Instalación en sistemas con Firebird $MAJOR.$MINOR
----------------------------------------

Adicionalmente al cambio de la estructura en disco 
(ODS), tenga en cuenta que el instalador no puede
detectar automáticamente si una versión Alfa o Beta
de Firebird $MAJOR.$MINOR se está ejecutando como servicio.
Por lo tanto, no recibirá ninguna advertencia hasta
que el instalador intente reemplazar el archivo
en uso. Esto puede evitarse mediante una desinstalación 
previa.


Nuevas características del instalador
-------------------------------------

o Si se encuentra el archivo firebird.conf el 
  instalador instalará el archivo de configuración
  por defecto como:

  firebird.conf.default.

  El archivo firebird.conf existente no se modificará.
  Esto busca asegurar que las restricciones de
  acceso se mantengan luego de una actualización.


o El instalador soporta ahora varios lenguajes,
  pero SOLO durante el proceso de instalación en sí.
  
  NOTA: Los lenguajes disponibles están determinados
  por la versión de Windows que utiliza. Esto
  significa que, por ejemplo, los usuarios del oeste
  de Europa no verán los lenguajes eslavos disponibles.
  

Problemas conocidos relacionados con la instalación
---------------------------------------------------

Ninguno conocido en este momento.


Desinstalación
--------------

o Es recomendable que este paquete sea correctamente
  desinstalado utilizando el desinstalador provisto, 
  que puede ser utilizado desde el Panel de Control.
  Como alternativa, puede ser desinstalado ejecutando
  unins000.exe directamente desde la carpeta de 
  instalación.

o Si Firebird se está ejecutando como una aplicación
  (en lugar de hacerlo como servicio), se recomienda
  detener el servidor manualmente antes de ejecutar
  el desinstalador, debido a que este último no puede
  detener una aplicación en ejecución.
  Si el servidor estuviera ejecutándose durante el 
  proceso de desinstalación, éste no se completará
  correctamente. Deberá eliminar los remanentes
  manualmente.
 
o El proceso de desinstalación dejará cuatro archivos
  en la carpeta de instalación:

  - aliases.conf
  - firebird.conf
  - firebird.log
  - security2.fdb

  Esto es intencional. Estos archivos son potencialmente
  modificables por los usuarios y podrían necesitarse
  si Firebird es reinstalado en el futuro. Pueden 
  eliminarse si no son necesarios.


Otras notas
-----------

  Firebird requiere WinSock2. Todas las plataformas
  Win32 deberían tenerlo, con excepción de Win95. 
  Durante la instalación se realiza una verificación
  de la librería Winsock2. Si la misma no se encuentra,
  la instalación fallará. Puede visitar el siguiente
  enlace:

    http://support.microsoft.com/default.aspx?scid=kb;EN-US;q177719

  para obtener información sobre cómo realizar la 
  actualización.
